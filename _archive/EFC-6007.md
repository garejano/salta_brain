**Regras gerais de listagem e consulta da carga**  

1. A listagem de professores deve incluir todos os professores que tenham alguma carga ativa no filtro informado, independente de seu status.
2. "Vago" deve ser tratado como um professor, para fins de listagem.
3. O professor (incluindo vago) é considerado com carga ativa se:
    1. Estiver em `ProfessorCarga` com `(DataSaida == null || SaidaTemporaria == true)`
    2. Estiver em `ProfessorCargaSubstituto` com `DataSaida == null`
4. O professor pode ter 3 tipos de carga, que devem ser exibidas em contadores distintos:
    1. **Titular:** `ProfessorCarga.DataSaida == null`
    2. **Licença/afastamento:** `ProfessorCarga.SaidaTemporaria == true` ("vago" não terá esse tipo)
    3. **Substituto:** `ProfessorCargaSubstituto.DataSaida == null`
5. Para consultar ou alterar a carga de um professor, ele deve ser selecionado.
6. Ao selecionar um professor, todas as suas cargas na rede devem ser exibidas, incluindo escolas que não estejam selecionadas no filtro, mas que o usuário tenha acesso.
7. Ao selecionar um professor, sua carga pode ser exibida de duas formas:
    1. Alocação geral: exibe os tempos agrupados por escola, com a data de entrada mais recente para aquela escola.
    2. Alocação por turma: exibe os tempos no nível mais baixo, de Turma+Disciplina, com a data de entrada correspondente.

  
  
**Regras de alteração de carga entre professores**  

1. Para alterar a carga de um professor, seleciona-se um motivo único de saída no topo da tela, junto com uma data de saída.
    1. Um professor não pode sair de uma turma antes de ter entrado nela. Logo, a menor data de saída possível deve ser a menor data de entrada listada na tela.
    2. Não é possível fazer lançamentos futuros. Logo, a maior data de saída possível é a data atual.
2. Ao selecionar uma data de saída, apenas as linhas que tenham data de entrada igual ou maior a saída selecionada podem ser alteradas.
3. Para remover uma carga, deve-se preencher o novo professor e a data de entrada do novo professor nessa mesma carga.
    1. Professores que estejam com status "Desligado" não devem aparecer na listagem.
    2. O próprio professor não deve aparecer na listagem. Ou seja, não é possível mover a carga de um professor para ele mesmo.
    3. Ou seja, caso uma carga deva continuar com o professor atual, não deve ser informada uma data de entrada e professor para ela.
    4. A data de entrada pode ser anterior a data de saída, mas não pode ser anterior a data de entrada do professor atual. Logo, a menor data de entrada possível deve ser a data de entrada do professor atual naquela carga.
    5. Não é possível fazer lançamentos futuros. Logo, a maior data de entrada possível é a data atual.
4. Caso a carga seja do tipo **licença/afastamento** (conforme definição nas regras de listagem, ponto 4.b), deve ser exibido em verde o nome do professor substituto naquela carga.
    1. Caso existam múltiplos substitutos para aquela carga (cenário possível apenas na aba de Alocação geral), deve ser exibido "Múltiplos professores" no lugar dos nomes dos substitutos.
5. Caso a carga seja do tipo **substituto** (conforme definição nas regras de listagem, ponto 4.c), deve ser exibido em roxo o nome do professor titular daquela carga.
    1. Caso existam múltiplos titulares para aquela carga (cenário possível apenas na aba de Alocação geral), deve ser exibido "Múltiplos professores" no lugar dos nomes dos titulares.
6. Existem 3 tipos de motivos de saída disponíveis:
    1. **"Normal"**, que pode ser aplicado para cargas em que o professor é **titular** ou **substituto**.
    2. **"Saída temporária"**, que pode ser aplicado para cargas em que o professor é **titular**.
    3. **"Retorno de saída temporária"**, que pode ser aplicado para cargas em que o professor está de **licença/afastamento**.
7. Ao selecionar um motivo de saída, só deve ser possível informar professores/datas (ou seja, alterações) para cargas que sejam do tipo correspondente.
8. Caso existam múltiplos tipos de carga para uma mesma linha (cenário possível apenas na aba de Alocação geral), no lugar dos campos de alteração deve se exibir uma indicação para o usuário utilizar a aba de alocação por turma.
9. Caso o motivo selecionado seja do tipo "**Retorno de saída temporária"**, os únicos professores disponíveis para seleção devem ser o próprio professor (para que volte a carga titular) e o professor que atualmente está como substituto.
    1. Ou seja, para casos de retorno a regra 3.b deve ser ignorada.
    2. Caso existam múltiplos substitutos para aquela carga (cenário possível apenas na aba de Alocação geral), no lugar dos campos de alteração deve se exibir uma indicação para o usuário utilizar a aba de alocação por turma.
    3. Caso o professor titular ou substituto estejam desligados, o mesmo professor não deve aparecer na listagem. Ou seja, a regra 3.a tem preferência.
10. Caso alguma data (de entrada ou saída) corresponda a um período passado (datas de uma folha já fechada/calculada), o sistema deve exibir um alerta para o usuário. Esse alerta é apenas informativo, para que o usuário confirme que está fazendo uma operação "fora do prazo".

Antes de salvar as alterações, o sistema deve exibir um resumo de todas as cargas que serão alteradas. Esse alerta é apenas informativo, para que o usuário confirme que realmente são aquelas alterações que deseja fazer.  
  
**Regras de alteração de carga de um professor para vago**  

1. Ao selecionar um novo professor para a carga, sempre deve ser exibida a opção de "Professor vago", como primeira opção da lista.

Caso o novo professor seja "vago", a data de entrada não deve ser informada manualmente, e sim ser salva automaticamente com a data de saída do professor atual.  
  
**Regras de alteração de carga de vago para um professor**  

1. Ao selecionar um novo professor para a carga, a opção "Professor vago" não deve ser exibida.
    1. Ou seja, não é possível mover de vago para vago, assim como não é possível mover de um professor para ele mesmo.
2. O único motivo de saída disponível deve ser "Entrada de novo professor", que deve ser tratado como um tipo "normal". Ou seja, é aplicável para cargas titulares e de substituto.
    1. Esse motivo de saída é exclusivo do vago, e não deve ser exibido quando for um professor real.
    2. Isso significa que o professor "vago" não poderá entrar de licença/afastamento.
3. Ao mover a carga de um professor vago para outro, a data de saída não deve ser informada manualmente, e sim ser salva automaticamente com a data de entrada do novo professor, para cada carga.
