# EFC-5976 — Configurador de Avaliações: Suporte a Escopos de Componente Formativo

---

## 1. Contexto e Motivação

O Novo Ensino Médio introduz **Itinerários Formativos (IF)** e **Componentes Formativos** — disciplinas com ciclos e avaliações distintos do currículo regular (FGB). Atualmente o Configurador de Avaliações mistura ambos no mesmo grid: disciplinas de itinerário têm poucas colunas preenchidas enquanto as regulares têm todas, tornando o configurador difícil de usar e o boletim de leitura ruim.

O objetivo deste card é **separar itinerários/componentes formativos em escopos próprios no configurador**, com colunas independentes e filtragem correta de disciplinas.

**Mapeamento para `ItinerarioFormativoCiclo.Id`** (IDs reais no banco — não são sequenciais):

| EscopoEnum | Valor | ItinerarioFormativoCiclo.Id | Nome no banco |
|------------|-------|-----------------------------|---------------|
| ComponenteFormativo_Anual | 4 | **1** | Anual |

| ComponenteFormativo_Trimestral | 6 | **4** | Trimestral |

Implementar o mapeamento via dicionário (não hardcode os inteiros diretamente nas queries):

```csharp
// Em FiltroManager ou classe utilitária
private static readonly Dictionary<EscopoEnum, int> MapaEscopoParaCicloId = new()
{
    { EscopoEnum.ComponenteFormativo_Anual,      1 },
    { EscopoEnum.ComponenteFormativo_Semestral,  2 },
    { EscopoEnum.ComponenteFormativo_Trimestral, 4 },
    { EscopoEnum.ComponenteFormativo_Bimestral,  5 }
};

private static int MapearEscopoParaCicloId(EscopoEnum escopo)
    => MapaEscopoParaCicloId[escopo];

private static bool EhEscopoCF(EscopoEnum escopo)
    => MapaEscopoParaCicloId.ContainsKey(escopo);
```

var mapaEscoposCF = [
    { IdCiclo: 1, Hash: 4, Descricao: 'Componente Formativo — Anual' },
    { IdCiclo: 2, Hash: 5, Descricao: 'Componente Formativo — Semestral' },
    { IdCiclo: 3, Hash: 6, Descricao: 'Componente Formativo — Trimestral' },
    { IdCiclo: 4, Hash: 7, Descricao: 'Componente Formativo — Bimestral' }
];

function AtualizarEscopos(agrupamento) {
    self.agrupamentoConfiguradoParaCF(agrupamento.PossuiItinerarioFormativoSeparadoNoBoletim);
    self.ciclosItinerarioFormativo(agrupamento.CiclosItinerarioFormativoExistentes || []);

    var escopos = [
        { Hash: 1, Descricao: 'Boletim Regular' },
        { Hash: 2, Descricao: 'Boletim Diversificado' },
        { Hash: 3, Descricao: 'Outras avaliações' }

    for (var cf of mapaEscoposCF) {
        if (self.ciclosItinerarioFormativo().indexOf(cf.IdCiclo) >= 0) {
            escopos.push(cf);
        }
    }

    self.escopos(escopos);
}

- **Regressão nos escopos 1–3:** A adição do filtro `ItinerarioFormativoCiclo == null` na query de `EstruturaAvaliacao` para escopos regulares é crítica — sem isso, colunas CF criadas no futuro apareceriam no boletim regular.
- **Retroatividade:** Avaliações já lançadas permanecem vinculadas ao escopo em que foram criadas. Não há impacto retroativo.
- **Compatibilidade de dados existentes:** `ItinerarioFormativo` sem ciclo preenchido deve ser tratado como `Semestral` (ou excluído dos novos escopos, dependendo da decisão de produto).
- **Geração de código:** O projeto usa `[GenerateDomain]` com geração automática de repositórios e factories. Após criar `ItinerarioFormativoCiclo`, executar o gerador para atualizar os arquivos em `Eleva.Portal/Generated/`.
- **Ciclos disponíveis no filtro:** Os escopos CF só aparecem na combo se houver `ItinerarioFormativo` com aquele ciclo configurado para o agrupamento. Evita poluição da lista em escolas que não usam NEM.
